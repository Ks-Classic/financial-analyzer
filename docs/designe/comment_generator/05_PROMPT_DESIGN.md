# 05. プロンプト設計

## 概要

AIコメント生成の品質はプロンプト設計に大きく依存する。
本ドキュメントでは、使用しているプロンプト構造と最適化ポイントを説明。

> **v2.0 更新 (2026-03-06)**: ソース限定制約、構造化出力対応、自動バリデーション連携を追加。

---

## プロンプト構造

### 3層構造

```
┌─────────────────────────────────────────────────────┐
│ システムプロンプト（全ページ共通）                  │
│ - AIの役割定義                                      │
│ - ソース限定制約 (Source-Only Constraint)            │
│ - 禁止表現ルール                                    │
│ - 80%カバールール                                   │
│ - 推測禁止ルール                                    │
│ - 数値正確性ルール                                  │
│ - 品質セルフチェック                                │
│ - 出力形式ルール                                    │
└─────────────────────────────────────────────────────┘
                        ＋
┌─────────────────────────────────────────────────────┐
│ ページプロンプト（ページ固有）                      │
│ - 分析の焦点                                        │
│ - 注目ポイント                                      │
│ - 特記事項                                          │
└─────────────────────────────────────────────────────┘
                        ＋
┌─────────────────────────────────────────────────────┐
│ 構造化出力指示（generate API のみ）                 │
│ - JSON出力フォーマット指示                          │
│ - 各フィールドの記入ルール                          │
│ - responseSchema による出力強制                     │
└─────────────────────────────────────────────────────┘
```

## デフォルトシステムプロンプト

```markdown
あなたは財務分析の専門家です。

【最重要ルール - ソース限定制約（Source-Only Constraint）】
あなたの知識ベースは「提供された画像・データ・前月コメント」のみです。
・画像やデータから直接読み取れる情報のみ使用してください
・一般的な経済知識、業界知識、季節要因に関する外部知識を一切使用しないでください
・データに存在しない情報について推測・補完・創作を絶対にしないでください
・「〜と考えられます」「〜の影響と思われます」等の推量表現は禁止です
・確認できない事項は「（担当者記入）要因は〇〇〇です。」と明記してください

【基本ルール】
・前月レポートのコメントのトーン・表現スタイルを踏襲してください
・当月の特徴的な変動やトピックがあれば言及してください
・他のページのデータから読み取れる事実を参照して要因を記載してください
・コメントは前月レポートと同程度の文字数にしてください
・数ヶ月の推移を見て傾向が変化した場合は、その点も考察してください

【表現ルール - 抽象的な形容表現の禁止】
・金額や数量の増減、比率の上下について、抽象的な形容表現を使用しないでください
・禁止表現の例: 著しく、僅かに、おおむね、概ね、大幅に、微増、微減、
  横ばい、やや、若干、顕著に、急激に、緩やかに
・代わりに具体的な数値（金額・変化率・個数など）で変動を表現してください
・例（NG）: 「売上高は大幅に増加しました」
・例（OK）: 「売上高は前月比1,490千円（7.6%）増加しました」

【変動要因の記載ルール - 80%カバー】
・全体の変動に対する寄与度を計算し、累計で80%に達するまでの要因のみ言及
・寄与度の小さい要因は省略してください
・例1）要因Aが80%、要因Bが20%寄与 → 要因Aのみに言及
・例2）要因Aが50%、要因Bが40%、要因Cが10%寄与 → 要因A、Bのみに言及

【要因分析ルール - 推測の禁止】
・変動の要因はチャート・表・データから直接読み取れる事実のみを記載
・季節要因、市場動向、需要変動など、データに明示されていない推測的な要因分析は禁止
・データから原因が特定できない場合は「（担当者記入）要因は〇〇〇です。」と記載
・例（NG）: 「季節的な需要変動の影響と考えられます」
・例（OK）: 「売上個数の減少（前月比200個減）が主因です」

【数値記載ルール - 正確性の担保】
・画像から読み取った数値は、桁区切り（カンマ）を含めて正確に転記
・変化率を記載する場合は、計算過程が正しいことを必ず検証
・計算例: 前月比 = (当月値 - 前月値) / 前月値 × 100
・複数の数値を比較する場合は、同じ単位で統一（千円 or 百万円）

【表記ルール - 半角・全角の一致】
・前月レポートやデータ中の固有名詞・商品名・コード名の表記を維持
・勝手に半角を全角に変換したり、全角を半角に変換しないこと
・例: 半角カタカナ「ﾐﾗｸﾙﾌｨｯﾄ」→ そのまま出力
・例: 半角括弧「ACC西(UCR)」→ そのまま出力

【出力形式 - PowerPoint用プレーンテキスト】
・マークダウン記法は一切使用禁止（#、**、__、-、*、> など全て禁止）
・プレーンテキストのみで出力
・箇条書きを使う場合は「・」（中黒）を先頭に使用
・段落の先頭は全角スペース1つでインデント
・強調したい部分は「」（かぎ括弧）で囲むか、文末に（重要）と付ける
・前月コメントと同じフォーマットを維持

【品質セルフチェック - 出力前に必ず確認】
出力する前に、以下の項目を内部で検証してください:
□ 禁止表現（著しく、大幅に、微増等）が含まれていないか
□ すべての数値は画像データから直接読み取ったものか
□ 推測・推量表現（〜と考えられます）が含まれていないか
□ 変動要因は80%カバールールを満たしているか
□ 前月コメントとトーン・文字数が大きく乖離していないか
```

## 構造化出力時の追加指示

`/api/comment/generate` エンドポイントでは、上記システムプロンプトに加え、以下の構造化出力指示が自動追加される。

```markdown
【構造化出力の注意事項】
・commentフィールドにPowerPoint用プレーンテキストのコメントを出力してください
・extracted_numbersには、コメントで言及した全ての数値を列挙してください
・variation_factorsには、変動要因を寄与度（%）付きで列挙してください
・sourceは必ず image_data / previous_comment / context_page のいずれかにしてください
・外部知識（季節要因、市場動向等）は source に含めないでください
・confidence_scoreは画像の読み取り精度に応じて0.0〜1.0で設定してください
・speculative_flagは推測的な内容を含む場合のみtrueにしてください（原則falseであるべき）
```

### 構造化出力スキーマ

```typescript
{
  comment: string;              // PowerPoint用プレーンテキスト
  extracted_numbers: Array<{    // 言及した数値の一覧
    label: string;              // 数値の説明（例: 売上高 前月比）
    value: number;              // 数値
    unit: string;               // 単位（千円、%、個 等）
    change_pct?: number | null; // 変化率（%）。不明の場合は0
  }>;
  variation_factors: Array<{    // 変動要因の一覧
    factor: string;             // 要因の説明
    contribution_pct: number;   // 全体変動に対する寄与度（%）
    source: 'image_data' | 'previous_comment' | 'context_page' | 'unknown';
  }>;
  confidence_score: number;     // 確信度 0.0〜1.0
  data_source: 'image_only' | 'image_and_context' | 'image_and_previous';
  speculative_flag: boolean;    // 推測的内容の有無
}
```

### 自動バリデーションとの連携

構造化出力により、以下のチェックが**プログラム的に自動実行**される:

| チェック項目 | 判定方法 | 従来（テキスト出力） | 構造化出力 |
|-------------|---------|-------------------|-----------|
| 禁止表現 | `comment` フィールドから文字列検索 | ❌ 手動確認 | ✅ 自動 |
| 推量表現 | 正規表現パターンマッチ | ❌ 手動確認 | ✅ 自動 |
| データソース違反 | `variation_factors.source` の値チェック | ❌ 確認不可 | ✅ 自動 |
| 80%カバールール | `contribution_pct` の累計計算 | ❌ 確認不可 | ✅ 自動 |
| 確信度 | `confidence_score` の閾値チェック | ❌ 確認不可 | ✅ 自動 |
| マークダウン混入 | `comment` フィールドから正規表現検出 | ❌ 手動確認 | ✅ 自動 |

## 一括生成時のプロンプト

### bulk-cache時のコンテキスト

```typescript
const contents = [
    {
        role: "user",
        parts: [
            { text: systemPrompt },
            // 各ページの画像とメタデータ
            ...pages.flatMap(page => [
                { text: `\n\n【ページ${page.pageNumber}: ${page.pageTitle}】` },
                { inlineData: { mimeType: "image/png", data: imageBase64 } },
                { text: `前月コメント: ${page.previousComment || 'なし'}` },
            ])
        ]
    }
];
```

### generate-fast時のプロンプト

```markdown
上記のキャッシュされた情報を参考に、
ページ{pageNumber}「{pageTitle}」のコメントを生成してください。

【出力形式】
- 200-300字程度
- 文章形式（箇条書き不可）
- 前月との比較を含める
- 変動要因を考察する

{pagePrompt}

コメントのみを出力してください。説明や前置きは不要です。
```

## 修正タイプ別プロンプト

| タイプ | プロンプト |
|--------|----------|
| `shorten` | コメントをより簡潔に要約してください。重要なポイントを維持しながら、文字数を半分程度に減らしてください。 |
| `detailed` | コメントをより詳細にしてください。数値分析を深堀りし、変動要因の考察を追加してください。 |
| `formal` | コメントをよりフォーマルな文体に変換してください。経営会議での報告に適した表現を使用してください。 |
| `casual` | コメントをより読みやすい文体に変換してください。専門用語を減らし、誰にでも理解しやすい表現にしてください。 |
| `custom` | ユーザーの自由記述による指示 |

## 最適化のポイント

### 1. 明確な役割定義

```markdown
❌ 悪い例: コメントを書いてください
✅ 良い例: あなたは財務分析の専門家です。経営層向けの報告用コメントを生成してください。
```

### 2. 具体的な出力形式

```markdown
❌ 悪い例: 適切な長さで
✅ 良い例: 200-300字程度で、文章形式（箇条書き不可）
```

### 3. 数値表現の強制

```markdown
❌ 悪い例: 「売上高は大幅に増加」（抽象的）
✅ 良い例: 「売上高は前月比1,490千円（7.6%）増加」（具体的数値）
```

### 4. ソース限定の明示

```markdown
❌ 悪い例: 変動の要因を分析してください
✅ 良い例: 画像・データから読み取れる事実のみで要因を記載。
          特定できない場合は「（担当者記入）要因は〇〇〇です。」と記載。
```

### 5. 不要な出力の抑制

```markdown
❌ → AIが「はい、わかりました。」などを出力
✅ 良い例: コメントのみを出力してください。説明や前置きは不要です。
```

## v1.0 → v2.0 主な変更点

| 項目 | v1.0 | v2.0 |
|------|------|------|
| ソース制約 | なし | ソース限定制約（外部知識使用禁止） |
| 禁止表現 | なし | 15語の抽象表現を明示的に禁止 |
| 推測禁止 | なし | 推量表現の正規表現パターンで自動検出 |
| 80%ルール | なし | 変動要因の寄与度80%カバーを強制 |
| 数値正確性 | 暗黙的 | 桁区切り転記・計算検証ルールを明示 |
| 表記一致 | なし | 半角/全角変換の禁止ルール |
| 品質チェック | なし | AI自身への5項目セルフチェック指示 |
| 構造化出力 | なし | responseSchema による JSON 出力強制 |
| バリデーション | 手動 | 7項目の自動バリデーション |
| モデル | `gemini-3-flash-preview` | `gemini-3.1-pro-preview`（生成）/ `gemini-3.1-flash-lite-preview`（抽出） |
| temperature | 0.7 | 0.3（再現性重視） |

## トラブルシューティング

| 問題 | 原因 | 対策 |
|------|------|------|
| 出力が長すぎる | 文字数制限が不明確 | 「200字以内」と明示 |
| 出力が抽象的 | 禁止表現ルール未適用 | 禁止表現リストで自動検出 |
| 前月比較がない | 前月データの活用不足 | 前月コメントを明示的に含める |
| 不要な前置き | 出力制御不足 | 「コメントのみ出力。説明不要」 |
| 推測的な要因分析 | ソース限定制約なし | ソース限定制約 + 推量表現チェック |
| 数値の誤り | 検証ルールなし | extracted_numbers で数値を構造化し検証可能に |
| マークダウン混入 | 出力形式不明確 | responseSchemaで制約 + 正規表現チェック |

## モデル設定

### コメント生成・修正（高品質推論モデル）
- **モデル**: `gemini-3.1-pro-preview`
- **temperature**: 0.3（再現性重視）
- **構造化出力**: `responseMimeType: 'application/json'` + `responseSchema`

### 抽出・OCR（高速モデル）
- **モデル**: `gemini-3.1-flash-lite-preview`
- **用途**: PDFテキスト抽出、画像データ読み取り
